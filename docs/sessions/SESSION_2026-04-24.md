## Fix 1 — CocoaPods objectVersion patch

Root cause: Xcode 16 writes objectVersion = 70 in project.pbxproj. CocoaPods 1.16.2 / xcodeproj 1.27.0 throws:
  ArgumentError - Unable to find compatibility version string for object version '70'

Fix:
  sed -i '' 's/objectVersion = 70/objectVersion = 60/' ios/KolasysAI.xcodeproj/project.pbxproj

Notes: Re-apply after any Xcode upgrade. Safe — Xcode ignores this field at build time.

## Fix 2 — WatchBridge file reference paths

Root cause: WatchBridge.swift and WatchBridge.m live in ios/KolasysAI/ (subdirectory) but pbxproj registered them with path = WatchBridge.swift (relative to ios/). Xcode looked for the files at ios/WatchBridge.swift and failed.

Error: Build input file cannot be found: '.../ios/WatchBridge.swift'

Fix:
  sed -i '' 's/path = WatchBridge\.swift;/path = KolasysAI\/WatchBridge.swift;/' ios/KolasysAI.xcodeproj/project.pbxproj
  sed -i '' 's/path = WatchBridge\.m;/path = KolasysAI\/WatchBridge.m;/' ios/KolasysAI.xcodeproj/project.pbxproj

## Fix 3 — Metro not running after build

Symptom: App shows "No script URL provided" after expo run:ios.
Cause: expo run:ios starts Metro but it stops when terminal closes.
Fix: Always keep a separate npx expo start terminal running. Shake phone → Reload.
